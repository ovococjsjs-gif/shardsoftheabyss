# shardsoftheabyss

Рабочая структура проекта книги **«Хроники Этериума — Осколки Бездны»**.

## Как этим пользоваться

### Если начинаем новую сессию работы
Читать по порядку:

1. `00-start-here/SESSION-BRIEF.md`
2. `02-writing-system/PRIMARY-WORK-INSTRUCTION.md`
3. `02-writing-system/HARD-BLOCKERS.md`
4. `02-writing-system/VOICE-SYSTEM.md`
5. `02-writing-system/WORKFLOW-AND-QUALITY-GATE.md`

Это **короткий обязательный пакет** перед любой работой над прозой.

---

## Структура

### `00-start-here/`
Короткие входные документы.

- `SESSION-BRIEF.md` — что сейчас считается актуальным, что читать первым, какие риски контролировать
- `PROJECT-MAP.md` — полная карта файлов и логики проекта

### `01-canon/`
Длинные канонические документы.

- `PROJECT-STATUS.md`
- `project-bible.md`
- `world-expansion-v0.1.md`

### `02-writing-system/`
Средний рабочий слой: голос, процесс, фильтры, примеры.

- `HARD-BLOCKERS.md`
- `VOICE-SYSTEM.md`
- `WORKFLOW-AND-QUALITY-GATE.md`
- `STYLE-EXAMPLES.md`

### `03-manuscript/`
Рукопись.

- `arc-01-v4-combined.md`
- `README.md`

### `90-archive/`
Старые исходные документы, дубли и архивные версии логики.

---

## Три уровня контекста

### Уровень 1 — короткий сессионный пакет
То, что нужно читать всегда:
- `00-start-here/SESSION-BRIEF.md`
- `02-writing-system/HARD-BLOCKERS.md`
- `02-writing-system/VOICE-SYSTEM.md`
- `02-writing-system/WORKFLOW-AND-QUALITY-GATE.md`

### Уровень 2 — рабочая система
То, что подключается по задаче:
- `00-start-here/PROJECT-MAP.md`
- `02-writing-system/STYLE-EXAMPLES.md`

### Уровень 3 — длинная база
То, что нужно для глубокой проверки и канона:
- `01-canon/*`
- `03-manuscript/*`
- `90-archive/*` при необходимости

---

## Главный принцип структуры

Проект больше не читается целиком перед каждой задачей.

Сначала используется **короткий входной пакет**, затем — нужный рабочий слой, и только потом при необходимости поднимается длинный канон или рукопись.
